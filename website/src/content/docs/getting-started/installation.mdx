## Docker

The fastest way to get started:

```bash
docker run -d --name frogdb -p 6379:6379 frogdb/frogdb:latest
```

With persistent storage:

```bash
docker run -d --name frogdb \
  -p 6379:6379 \
  -v frogdb-data:/data \
  frogdb/frogdb:latest
```

Docker Compose:

```yaml
services:
  frogdb:
    image: frogdb/frogdb:latest
    ports:
      - "6379:6379"
    volumes:
      - frogdb-data:/data
    environment:
      FROGDB_SERVER__BIND: "0.0.0.0"
      FROGDB_PERSISTENCE__DATA_DIR: /data
    healthcheck:
      test: ["CMD", "redis-cli", "-p", "6379", "PING"]
      interval: 5s
      retries: 10

volumes:
  frogdb-data:
```

## Debian/Ubuntu Package

```bash
# Download the latest .deb package from GitHub Releases
sudo dpkg -i frogdb_*.deb

# The server is managed via systemd
sudo systemctl start frogdb-server
sudo systemctl enable frogdb-server
```

The package installs to `/usr/bin/frogdb-server` with data at `/var/lib/frogdb/`.

## Build from Source

<details>
<summary>For development or custom builds</summary>

Requires Rust 1.75+ ([rustup.rs](https://rustup.rs/)) and RocksDB dependencies.

**macOS:**
```bash
brew install rocksdb snappy lz4 zstd
```

**Ubuntu/Debian:**
```bash
sudo apt-get install -y build-essential librocksdb-dev libsnappy-dev liblz4-dev libzstd-dev
```

**Arch Linux:**
```bash
sudo pacman -S rocksdb snappy lz4 zstd
```

Build:
```bash
git clone https://github.com/frogdb/frogdb.git
cd frogdb
cargo build --release
# Binary at ./target/release/frogdb-server
```

</details>
