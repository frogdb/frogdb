FrogDB is wire-compatible with RESP2 and RESP3, so you can use your standard Redis client.

## Start the Server

<Tabs>
  <TabItem label="Binary" icon="seti:shell">
    ```bash
    # Start with defaults (port 6379, persistence enabled)
    frogdb-server

    # Or with a config file
    frogdb-server --config frogdb.toml

    # Auto-detect shards, bind to all interfaces
    frogdb-server -b 0.0.0.0 -s auto
    ```
  </TabItem>
  <TabItem label="Docker" icon="seti:docker">
    ```bash
    docker run -d --name frogdb \
      -p 6379:6379 \
      -v frogdb-data:/data \
      frogdb/frogdb:latest
    ```
  </TabItem>
  <TabItem label="Docker Compose" icon="seti:docker">
    ```yaml
    services:
      frogdb:
        image: frogdb/frogdb:latest
        ports:
          - "6379:6379"
        volumes:
          - frogdb-data:/data
        environment:
          FROGDB_SERVER__BIND: "0.0.0.0"
          FROGDB_PERSISTENCE__DATA_DIR: /data
        healthcheck:
          test: ["CMD", "redis-cli", "-p", "6379", "PING"]
          interval: 5s
          retries: 10

    volumes:
      frogdb-data:
    ```

    ```bash
    docker compose up -d
    ```
  </TabItem>
</Tabs>

## Connect

```bash
redis-cli -p 6379
```

```redis
127.0.0.1:6379> SET greeting "Hello from FrogDB!"
OK
127.0.0.1:6379> GET greeting
"Hello from FrogDB!"
127.0.0.1:6379> INFO server
# Server info including version
```

## Configuration

Generate a default config file:

```bash
frogdb-server --generate-config > frogdb.toml
```

Common options:

```toml
[server]
bind = "0.0.0.0"        # Listen on all interfaces
port = 6379
num-shards = 0           # 0 = auto-detect CPU cores

[persistence]
enabled = true
data-dir = "/var/lib/frogdb"
durability-mode = "periodic"  # async, periodic, sync
sync-interval-ms = 1000

[memory]
maxmemory = 4294967296   # 4 GB
maxmemory-policy = "allkeys-lru"
```

See [Configuration Reference](/reference/configuration/) for all options, or the [Reference Config File](/reference/reference-config/) for a complete example with defaults.
