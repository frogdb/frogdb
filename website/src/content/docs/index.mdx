## Clustering & Replication

FrogDB scales horizontally with hash-slot sharding across a cluster of nodes, coordinated by an external orchestrator rather than a Redis-style gossip protocol.

- **16384 hash slots** (`CRC16(key) % 16384`), with slot ranges assigned to nodes and rebalanced by the orchestrator
- **Raft consensus** for cluster state — deterministic leader election, no split-brain
- **Read replicas** — full copies of a primary's dataset that can serve stale reads and are promoted on automatic failover
- **Online slot migration** — nodes serve `-ASK`/`-MOVED` redirects during rebalancing, so the cluster keeps handling traffic
- **Cross-shard transactions** via [VLL](/architecture/vll/) — atomic multi-key operations across internal shards without a two-phase commit protocol

A custom Kubernetes controller for FrogDB is in development: declarative cluster provisioning, scaling, failover, and rolling upgrades managed natively from Kubernetes.
:::

## Redis 8.x Data Structures

FrogDB supports the full Redis 8.x feature set:

- **Core types** — Strings, Lists, Sets, Sorted Sets, Hashes, Streams
- **Bitmaps & Bitfields** — BITCOUNT, BITOP, BITPOS, BITFIELD
- **JSON** — RedisJSON-compatible document storage with JSONPath
- **Time Series** — Gorilla-compressed time series with aggregation and downsampling
- **Vector Sets** — Approximate nearest-neighbor search
- **Probabilistic** — Bloom filters, Cuckoo filters, HyperLogLog, Count-Min Sketch, Top-K, T-Digest
- **Geospatial** — Geohash indexing and distance queries
- **Pub/Sub** — Channel and pattern-based publish/subscribe, event sourcing
- **Scripting & Transactions** — Lua scripting, MULTI/EXEC
- **Search** — Full text search, secondary indexing, vector search, aggregations

## Testing

FrogDB is verified with deterministic concurrency simulation, distributed-systems testing, and a ported Redis regression suite that runs on every commit.

- **Shuttle and Turmoil** for deterministic concurrency and distributed simulation testing
- **Jepsen verification** of linearizability and serializability
- **Fuzz testing** (`cargo-fuzz`) and property-based tests (`proptest`)
- **2,232 ported Redis regression tests** from the Redis 8.6.0 TCL suite — see the [Redis Test Suite compatibility report](/compatibility/redis-test-suite/)

## Quick Example

```bash
# Start FrogDB
frogdb-server --port 6379

# Connect with redis-cli
redis-cli -p 6379

# Use familiar Redis commands
127.0.0.1:6379> SET greeting "Hello from FrogDB!"
OK
127.0.0.1:6379> GET greeting
"Hello from FrogDB!"
```
