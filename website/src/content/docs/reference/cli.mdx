## frogdb-server

The main FrogDB server process.

```bash
frogdb-server [OPTIONS]
```

### Options

| Option | Short | Default | Description |
|---|---|---|---|
| `--config <FILE>` | `-c` | | Path to configuration file |
| `--bind <ADDR>` | `-b` | `127.0.0.1` | Bind address |
| `--port <PORT>` | `-p` | `6379` | Listen port |
| `--shards <N>` | `-s` | `1` | Number of shards (`auto` = num CPUs) |
| `--log-level <LEVEL>` | `-l` | `info` | Log level: trace, debug, info, warn, error |
| `--log-format <FORMAT>` | | `pretty` | Log format: pretty, json |
| `--admin-bind <ADDR>` | | | Admin bind address |
| `--admin-port <PORT>` | | | Admin port (implies admin.enabled=true) |
| `--http-bind <ADDR>` | | | HTTP server bind address |
| `--http-port <PORT>` | | | HTTP server port |
| `--http-token <TOKEN>` | | | Bearer token for protected HTTP endpoints |
| `--tls-enabled` | | | Enable TLS |
| `--tls-cert-file <FILE>` | | | TLS certificate file (PEM) |
| `--tls-key-file <FILE>` | | | TLS private key file (PEM) |
| `--tls-ca-file <FILE>` | | | TLS CA certificate file (PEM) |
| `--tls-port <PORT>` | | | TLS listen port |
| `--tls-require-client-cert <MODE>` | | | Client cert mode: none, optional, required |
| `--tls-replication` | | | Encrypt replication connections with TLS |
| `--tls-cluster` | | | Encrypt cluster bus connections with TLS |
| `--generate-config` | | | Print default configuration and exit |
| `--intrinsic-latency <SECS>` | | | Run latency test for N seconds and exit |
| `--startup-latency-check` | | | Run latency check before accepting connections |

### Examples

```bash
# Start with defaults
frogdb-server

# Start with config file
frogdb-server --config /etc/frogdb/frogdb.toml

# Auto-detect shards, bind to all interfaces
frogdb-server -b 0.0.0.0 -s auto

# Generate default config
frogdb-server --generate-config > frogdb.toml

# Enable TLS
frogdb-server --tls-enabled --tls-cert-file server.crt --tls-key-file server.key
```

### Environment Variables

All configuration fields can be overridden via environment variables with the `FROGDB_` prefix. Use double underscores (`__`) to separate nested keys:

```bash
FROGDB_SERVER__BIND=0.0.0.0
FROGDB_SERVER__PORT=6380
FROGDB_PERSISTENCE__DURABILITY_MODE=sync
FROGDB_MEMORY__MAXMEMORY=4294967296
```

## frogctl

Operational CLI tool for managing FrogDB.

```bash
frogctl [GLOBAL OPTIONS] <COMMAND>
```

### Global Options

| Option | Short | Default | Description |
|---|---|---|---|
| `--host <HOST>` | `-H` | `127.0.0.1` | Server hostname or IP |
| `--port <PORT>` | `-p` | `6379` | RESP port |
| `--auth <PASSWORD>` | `-a` | | Password (AUTH) |
| `--user <USER>` | `-u` | | ACL username |
| `--tls` | | | Enable TLS |
| `--tls-cert <PATH>` | | | Client certificate (mTLS) |
| `--tls-key <PATH>` | | | Client private key (mTLS) |
| `--tls-ca <PATH>` | | | CA certificate |
| `--admin-url <URL>` | | | Admin HTTP base URL |
| `--metrics-url <URL>` | | | Metrics HTTP base URL |
| `--output <FORMAT>` | `-o` | `table` | Output: table, json, raw |
| `--no-color` | | | Disable ANSI colors |

### Commands

| Command | Description |
|---|---|
| `health` | Health checking |
| `stat` | Real-time monitoring dashboard |
| `config` | Configuration management |
| `cluster` | Cluster operations |
| `replication` | Replication management |
| `debug` | Diagnostics and debugging |
| `backup` | Backup and restore |
| `data` | Data utilities |
| `exec` | Execute a raw Redis command |
| `acl` | ACL user management |
| `client` | Client connection management |
| `scan` | Scan keyspace with optional enrichment |
| `watch` | Stream live commands (MONITOR) |
| `subscribe` | Subscribe to Pub/Sub channels or patterns |
| `search` | RediSearch operations (FT.* commands) |
| `benchmark` | Lightweight built-in benchmark |
| `upgrade` | Rolling upgrade management |

### Examples

```bash
# Health check
frogctl health

# Real-time stats dashboard
frogctl stat

# Execute a raw command
frogctl exec GET mykey

# Scan keys matching a pattern
frogctl scan --match "user:*"

# Watch live commands
frogctl watch

# Cluster info
frogctl cluster info

# Backup
frogctl backup snapshot

# Output as JSON
frogctl -o json health
```
