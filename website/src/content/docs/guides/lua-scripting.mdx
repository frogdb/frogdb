FrogDB supports Lua scripting for executing atomic multi-key operations. Scripts run on the server side, eliminating round-trip latency for complex operations.

## Why Use Lua Scripts?

- **Atomicity** — Scripts execute without interruption
- **Performance** — Reduce network round-trips
- **Complex Logic** — Implement operations not possible with single commands

## EVAL Command

Execute a Lua script:

```redis
EVAL script numkeys [key ...] [arg ...]
```

Example — atomic increment with limit:

```redis
EVAL "
  local current = tonumber(redis.call('GET', KEYS[1]) or 0)
  if current >= tonumber(ARGV[1]) then
    return nil
  end
  return redis.call('INCR', KEYS[1])
" 1 counter 100
```

Parameters:
- `script` — Lua code to execute
- `numkeys` — Number of keys that follow
- `KEYS[n]` — Keys accessed by the script (1-indexed)
- `ARGV[n]` — Additional arguments (1-indexed)

## Script Caching with EVALSHA

For frequently-used scripts, cache them to avoid sending the full script each time:

```redis
# Load script and get SHA1 hash
SCRIPT LOAD "return redis.call('GET', KEYS[1])"
# "a5260dd66ce02462c5b5f0e0c543e63e7e9f7b1c"

# Execute by hash
EVALSHA a5260dd66ce02462c5b5f0e0c543e63e7e9f7b1c 1 mykey
```

### Check if Script Exists

```redis
SCRIPT EXISTS a5260dd66ce02462c5b5f0e0c543e63e7e9f7b1c
# 1) (integer) 1
```

## Redis Commands in Lua

Call Redis commands using `redis.call()` or `redis.pcall()`:

```lua
-- redis.call() - errors propagate up
local value = redis.call('GET', KEYS[1])

-- redis.pcall() - errors are caught
local result, err = redis.pcall('GET', KEYS[1])
if err then
  return {err = err}
end
```

## Common Patterns

### Rate Limiting

```redis
EVAL "
  local key = KEYS[1]
  local limit = tonumber(ARGV[1])
  local window = tonumber(ARGV[2])

  local current = tonumber(redis.call('GET', key) or 0)

  if current >= limit then
    return 0
  end

  current = redis.call('INCR', key)
  if current == 1 then
    redis.call('EXPIRE', key, window)
  end

  return limit - current
" 1 ratelimit:user:123 100 60
```

### Compare and Set

```redis
EVAL "
  local key = KEYS[1]
  local expected = ARGV[1]
  local new_value = ARGV[2]

  local current = redis.call('GET', key)
  if current == expected then
    redis.call('SET', key, new_value)
    return 1
  end
  return 0
" 1 mykey "old_value" "new_value"
```

### Conditional Update

```redis
EVAL "
  local key = KEYS[1]
  local field = ARGV[1]
  local value = ARGV[2]

  -- Only update if key exists
  if redis.call('EXISTS', key) == 1 then
    return redis.call('HSET', key, field, value)
  end
  return nil
" 1 user:123 status "active"
```

### Atomic Transfer

```redis
EVAL "
  local from = KEYS[1]
  local to = KEYS[2]
  local amount = tonumber(ARGV[1])

  local balance = tonumber(redis.call('GET', from) or 0)
  if balance < amount then
    return {err = 'insufficient funds'}
  end

  redis.call('DECRBY', from, amount)
  redis.call('INCRBY', to, amount)

  return 'OK'
" 2 account:alice account:bob 100
```

### Leaderboard Update

```redis
EVAL "
  local key = KEYS[1]
  local member = ARGV[1]
  local score_delta = tonumber(ARGV[2])

  local current = redis.call('ZSCORE', key, member) or 0
  local new_score = current + score_delta

  redis.call('ZADD', key, new_score, member)
  return redis.call('ZRANK', key, member)
" 1 leaderboard player1 10
```

## Data Types

### Lua to Redis

| Lua Type | Redis Type |
|----------|------------|
| number | Integer |
| string | Bulk string |
| table (array) | Array |
| table (map) | Map (RESP3) |
| boolean false, nil | Null |
| boolean true | Integer 1 |

### Redis to Lua

| Redis Type | Lua Type |
|------------|----------|
| Integer | number |
| Bulk string | string |
| Array | table |
| Null | nil |
| Error | table with `err` field |

## Script Management

```redis
# List loaded scripts
SCRIPT EXISTS sha1 [sha1 ...]

# Flush script cache
SCRIPT FLUSH

# Kill running script (if taking too long)
SCRIPT KILL

# Check if scripts are enabled
SCRIPT DEBUG
```

## Best Practices

### 1. Always Declare Keys

Pass all accessed keys through `KEYS[]`, not as arguments:

```lua
-- Good
local val = redis.call('GET', KEYS[1])

-- Bad - breaks in cluster mode
local val = redis.call('GET', 'hardcoded:key')
```

### 2. Keep Scripts Short

Long-running scripts block other operations. Break complex logic into smaller scripts or use background processing.

### 3. Handle Errors

Use `redis.pcall()` for commands that might fail:

```lua
local ok, result = pcall(function()
  return redis.call('HGET', KEYS[1], ARGV[1])
end)

if not ok then
  return {err = result}
end
```

### 4. Use EVALSHA in Production

```python
# Python example
script = """
  return redis.call('GET', KEYS[1])
"""
sha = client.script_load(script)

# Use EVALSHA for subsequent calls
result = client.evalsha(sha, 1, 'mykey')
```

## Cluster Considerations

In cluster mode, all keys must be in the same hash slot:

```redis
# Use hash tags to ensure same slot
EVAL "
  redis.call('SET', KEYS[1], ARGV[1])
  redis.call('SET', KEYS[2], ARGV[2])
" 2 {user:1}:name {user:1}:email "Alice" "alice@example.com"
```

## Debugging Scripts

Enable script debugging (development only):

```redis
SCRIPT DEBUG YES

# Run script with debugger attached
EVAL "..." 1 key1
```
