FrogDB supports asynchronous primary-replica replication for high availability and read scaling.

## Overview

In a replication setup:

- **Primary** — Handles all writes, propagates changes to replicas
- **Replica** — Receives changes from primary, handles read traffic

```
┌─────────────┐     writes     ┌─────────────┐
│   Primary   │ ─────────────▶ │   Replica   │
│  (read/write)│               │ (read-only) │
└─────────────┘                └─────────────┘
```

## Setting Up Replication

### Configure the Primary

The primary needs no special configuration — it's the default role:

```toml
# primary.toml
[server]
bind = "0.0.0.0"
port = 6379

[persistence]
enabled = true
data_dir = "/var/lib/frogdb"
mode = "hybrid"
```

### Configure Replicas

Each replica connects to the primary:

```toml
# replica.toml
[server]
bind = "0.0.0.0"
port = 6380  # Different port if on same host

[replication]
role = "replica"
primary_host = "primary.example.com"
primary_port = 6379

[persistence]
enabled = true
data_dir = "/var/lib/frogdb-replica"
mode = "periodic"
```

Start the replica:

```bash
frogdb-server --config replica.toml
```

### Runtime Configuration

You can also configure replication at runtime:

```redis
# Make this node a replica of primary at 192.168.1.10:6379
REPLICAOF 192.168.1.10 6379

# Promote replica to primary
REPLICAOF NO ONE
```

## Initial Synchronization

When a replica connects:

1. **Full sync** — Primary sends a snapshot of all data
2. **Partial sync** — Primary streams ongoing writes
3. **Caught up** — Replica is in sync

Monitor sync progress:

```redis
# On replica
INFO replication
# role:replica
# master_host:primary.example.com
# master_port:6379
# master_link_status:up
# master_sync_in_progress:0
# slave_repl_offset:12345
```

## Read Scaling

Replicas can serve read traffic to scale read capacity:

```python
# Application code - direct reads to replicas
import redis

primary = redis.Redis(host='primary', port=6379)
replica = redis.Redis(host='replica', port=6380)

# Writes go to primary
primary.set('key', 'value')

# Reads can go to replica
value = replica.get('key')
```

### Stale Reads

Replication is asynchronous, so replicas may be slightly behind. Configure your application to tolerate this or use read-your-writes consistency patterns.

## Replica Read-Only Mode

By default, replicas reject write commands:

```redis
# On replica
SET key value
# (error) READONLY You can't write against a read only replica
```

## Monitoring Replication

### On Primary

```redis
INFO replication
# role:master
# connected_slaves:2
# slave0:ip=192.168.1.11,port=6380,state=online,offset=12345,lag=0
# slave1:ip=192.168.1.12,port=6380,state=online,offset=12340,lag=1
```

### On Replica

```redis
INFO replication
# role:slave
# master_host:192.168.1.10
# master_port:6379
# master_link_status:up
# master_last_io_seconds_ago:1
# slave_repl_offset:12345
# slave_read_only:1
```

### Key Metrics

| Metric | Description | Alert When |
|--------|-------------|------------|
| `master_link_status` | Connection to primary | `down` |
| `master_last_io_seconds_ago` | Seconds since last data | > 30 |
| `slave_repl_offset` | Replication position | Falling behind |

## Failover

### Manual Failover

If the primary fails:

1. **Stop traffic** to the failed primary
2. **Promote a replica** to primary:
   ```redis
   REPLICAOF NO ONE
   ```
3. **Reconfigure other replicas** to follow the new primary:
   ```redis
   REPLICAOF new-primary.example.com 6379
   ```
4. **Update application** connection strings

### Automated Failover

For automated failover, use FrogDB clustering mode or an external sentinel system.

## Replication Topology Examples

### Single Replica

```
Primary ──▶ Replica
```

Simple HA setup. Replica can be promoted if primary fails.

### Multiple Replicas

```
         ┌──▶ Replica 1
Primary ─┼──▶ Replica 2
         └──▶ Replica 3
```

Scale reads across multiple replicas.

### Chained Replication

```
Primary ──▶ Replica 1 ──▶ Replica 2
```

Reduce load on primary by having replicas replicate from other replicas.

```toml
# replica2.toml - replicates from replica1
[replication]
role = "replica"
primary_host = "replica1.example.com"
primary_port = 6380
```
